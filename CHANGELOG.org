#+title: Changelog

** 0.1.13

- Classify =block= agenda entries as events, not TODOs
  - org-agenda emits =block= for ranged =<a>--<b>= timestamps (e.g. a calendar event that spans hours or days). Previously =isEventEntry= only matched =timestamp= and =sexp=, so block-type events fell through to the TODO list with empty state/priority pills. Now they route to =EventBanners= alongside the other calendar-style entries.

** 0.1.12

- Render active timestamps in notes as formatted chips instead of raw =<...>= text
  - Backend =eav--parse-timestamp= now delegates to =org-element-timestamp-parser=, returning structured start/end components, range type, repeater, and warning fields
  - =<a>--<b>= ranges (e.g. Game Night spanning midnight) come back as a single entry with both endpoints -- not two separate timestamps
  - =eav-get-heading-notes= also returns parsed =activeTimestamps= so the frontend can render chips by looking up each =raw= occurrence
  - =NotesRenderer= swaps each =raw= string in the body with a =TimestampChip= that formats via the browser's locale
- Test infrastructure
  - Elisp fixture-based tests (=elisp/tests/fixtures/{calendar-events,nested-lists}.org=) covering every timestamp flavor and every checklist bullet type (=-=, =+=, =*=, =1.=, =1)=, mixed, three-level). =npm test= runs 33 ERT tests
  - Playwright end-to-end tests (=npm run test:e2e=) on desktop Chromium and mobile WebKit (iPhone 13) that exercise the Game Night rendering fix against a real browser with mocked =/api/*=

** 0.1.11

- Nested org checklists render with hierarchy (indentation per nesting level)
  - =NotesRenderer= tracks each item's source indent and maps distinct indents to depths per block
- Parent checkbox states aggregate from descendants, matching org's native behavior
  - After saving notes, elisp helper =eav--propagate-checkbox-parents= runs =org-list-struct-fix-box= on each list in the heading body so parents become =[X]= when all children are checked, =[-]= when some are checked or in progress
  - =eav-set-heading-notes= now returns the propagated notes; the UI updates local state with the server's final version

** 0.1.10

- Fix checklist parsing for all org bullet types
  - =+= and =*= bullets now parsed as checklist items (not just =-=)
  - Ordered bullets (=1.=, =a)=) with checkboxes also supported
  - Sub-items (=-= lines under a =+ [ ]= item) grouped under their parent
  - Multi-line checklist items render sub-text indented below
  - Checkbox toggle works for all bullet types

** 0.1.9

- Persist sort, group-by, and show-done settings in =localStorage=
  - Restored on page load so preferences survive refresh

** 0.1.8

- Mobile UX improvements
  - Long-press (500ms) on tasks opens context menu (replaces right-click on mobile)
  - Increased row padding, pill sizes, and font sizes on mobile for better touch targets
  - =select-none= prevents text selection during long-press

** 0.1.7

- Fix false "Connection Error" screen on transient emacsclient failures
  - Clock status polling is now best-effort; failure doesn't take down the app
  - After initial load, refresh failures preserve existing data instead of showing error screen
  - Fixed infinite retry loop caused by =error= in =useCallback= dependency array
- Date picker auto-confirms when time input loses focus (no need to press the confirm button)
- Mutation failures (stale positions) now trigger a data refresh for recovery

** 0.1.6

- Right-click context menu on tasks with "Edit title" and "Refile" options
  - Edit title: inline text input using org's =org-edit-headline= (preserves TODO state, priority, tags)
  - Refile: searchable modal with all =org-refile-get-targets= headings, type to filter
  - Uses org's own =org-refile= with =rfloc= parameter for correct refile behavior

** 0.1.5

- Fix capture: use =cl-letf= to advise =org-capture--position-cursor= instead of string replacement
  - Lets org's full template expansion pipeline run unmodified
  - Title is inserted at =%?= position by the advised cursor function
- Optional priority, scheduled date, and deadline when capturing
  - Inline pickers in capture form (priority A-D, date with quick options + time)
  - Applied to the captured entry via =org-capture-last-stored-marker= after finalize

** 0.1.4

- Org capture from the web UI
  - Reads =org-capture-templates= from Emacs and presents them in a modal (=Cmd+N= or sidebar =+= button)
  - Supports =entry= type templates with static file/headline targets
  - Templates with function targets or bodies shown as "Emacs only"
  - Runs org-capture non-interactively with =:immediate-finish=; all =%u=, =%t=, =%<...>= expansions handled by org
  - "+" button in sidebar header for quick access

** 0.1.3

- Secondary sort by scheduled/deadline time within primary sort groups
  - When sorting by priority, tasks with the same priority are sub-sorted by their scheduled time
  - Works across all sort modes (priority, state, category, deadline)

** 0.1.2

- Mobile-friendly responsive layout
  - Sidebar becomes a slide-over drawer with backdrop on small screens
  - Header controls (sort, group) wrap to a second row on narrow viewports
  - Reduced padding and expanded-panel indent on mobile
  - Metadata rows and date pickers wrap gracefully
- Production server mode: Express serves built frontend (=npm start=)
  - Single-process deployment on port 3001, no separate Vite dev server needed
- launchd service plist for running as a background macOS service
- Safe-area / notch support for iOS devices (=viewport-fit=cover=, =env(safe-area-inset-*)=)
- Use =100dvh= for proper mobile browser viewport height
- Touch-action: manipulation on buttons to eliminate tap delay
- Fix Express 5 catch-all route syntax (=*= -> =/{*path}=) for production static serving

** 0.1.1

- Fix checklist rendering: support all three org checkbox states (=[ ]=, =[-]=, =[X]=)
- In-progress =[-]= items render as orange checkboxes with a dash, italic text
- Progress bar shows orange segment for in-progress items alongside green for checked
- Clicking a checkbox cycles: unchecked -> in-progress -> checked -> unchecked

** 0.1

Initial feature-complete release.

- Today/Upcoming/All Tasks views using org-agenda's own machinery
- Clickable TODO states, priorities, scheduled/deadline dates with calendar+time picker
- Inline notes viewing/editing with interactive org checklists
- Org clocking (clock-in, clock-out, live timer banner)
- Multi-level grouping (agenda type, priority, category, state) with collapsible headers
- Calendar event banners, org markup rendering (bold, italic, code, links)
- Light/dark/auto theme switching with proper CSS variable system
- Collapsible sidebar with keyboard shortcut (=Cmd+\=)
