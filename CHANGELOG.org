** 0.4.0

The Rust daemon. Reads land on an in-memory index in ~2 ms; mutations and
the residual semantic reads proxy to Emacs over one persistent UNIX socket
instead of an =emacsclient --eval= per call. Live updates push over SSE so
the UI refreshes on file save and TODO state change without polling.

This is the *dogfood* release on the new architecture. v1.0.0 will land
once Express has been removed and the dogfood week is clean.

*** =eavd= -- the Rust daemon

New Cargo workspace under =daemon/= with seven crates (=eav-core=,
=eav-parse=, =eav-index=, =eav-agenda=, =eav-bridge=, =eav-server=,
=eavd=). The HTTP surface mirrors the Express routes 1:1; the
agenda evaluator hits 30/30 days byte-identical against the captured
=/api/agenda/day/<date>= corpus, and =/api/tasks= matches Express on
active and all-tasks counts. Cold-start <50 ms with a SQLite snapshot at
=$XDG_CACHE_HOME/eavd/snapshot.sqlite=.

- Parser uses =orgize 0.10.x= for headline structure plus a small regex
  fallback for the deadline-repeater syntax (=+1w/2w=) that orgize
  currently rejects (=org-element.el:4445-4448=). Properties, tag
  inheritance, file-local =#+TODO=/=#+CATEGORY=/=#+PROPERTY= overrides,
  malformed planning-after-CLOCK lines all handled.
- File watcher debounces 100 ms and suppresses fsnotify events that
  =eavd= itself triggered via the bridge (avoids redundant reindex).
- 24 fixture-driven tests grounded in =org-emphasis-regexp-components=
  and =org-element-timestamp-parser=, plus a regression test for the
  symlink-vs-canonical snapshot bug noted below.

*** =elisp/eav-bridge.el= -- in-Emacs UNIX-socket dispatcher

A thin (~480 LOC) elisp file that opens a UNIX socket via
=make-network-process=, reads length-prefixed JSON frames, and dispatches
to existing =eav.el= functions. Pushes =after-save=,
=todo-state-changed=, and clock-event frames so =eavd= can re-index and
publish SSE without polling. =eav.el= itself is unchanged -- the bridge
is purely a transport layer.

=eavd= auto-loads the bridge on first connect via =emacsclient --eval=
when bundled inside =Agenda.app=, so end-users never load it manually.

*** Mac app -- helper-process model

=apps/macos/EmacsAgendaViewer/Networking/DaemonHost.swift= spawns the
bundled =eavd= on app launch and terminates it on quit. The Run-Script
build phase in =project.yml= cargo-builds =eavd= and copies it +
=eav.el= + =eav-bridge.el= into =Contents/Resources/=. Closing the app
stops the daemon; reopening starts a fresh one.

=Networking/EventSubscriber.swift= consumes the SSE channel and feeds
=TasksStore.invalidate(file:)= / =invalidate(taskId:)= /
=invalidateConfig= so a save in Emacs (or an external tool) refreshes
the views without manual reload.

=AppDelegate= polls =/api/debug= until the helper answers, *then* writes
the saved =serverURL= -- so =RootView.task(id:serverURLString)= only
fires once the daemon's actually listening. Removes the
"Couldn't load" race we hit on first cold start.

*** Web frontend

=src/hooks/useDaemonEvents.ts= subscribes to =/api/events= via
=EventSource= and triggers a full refresh on each event, with a
3-strikes-and-out reconnect policy so it falls silent against the
legacy Express server (which has no SSE endpoint). =vite.config.ts=
proxy now points at =:3002= by default; override via =VITE_API_TARGET=.

*** Bundling and deployment

- =scripts/build-eavd-universal.sh= -- cross-compile =eavd= for both
  Apple architectures and =lipo=-fuse them. Used by Xcode's Run-Script
  phase for =Agenda.app=.
- =scripts/install-daemon.sh= -- install =eavd= as a launchd plist
  (macOS) or systemd user unit (Linux), for headless deployments where
  there is no Mac app to babysit the daemon. Sets =PATH= explicitly
  because launchd-Aqua services don't inherit Homebrew on the search
  list. Accepts =--port=, =--host=, =--static-dir= for the SPA.
- =deploy-web-eavd.sh= -- rsync the repo to the headless host, build
  =eavd= + the SPA there, retire the legacy
  =com.hermitsage.emacs-agenda-viewer= plist, install the new
  =com.hermitsage.eavd= one. Smoke-tests =$REMOTE_URL/api/config= and
  the SPA root.
- =server/= and =deploy-web.sh= remain available for fallback. v1.0.0
  removes them.

*** Snapshot canonicalize fix

Files arriving via the bridge config (e.g. =~/iCloud/Notes/foo.org=)
were being keyed in =Index::by_file= by their input path, while the
SQLite snapshot stored canonicalised paths (=~/Library/Mobile
Documents/.../foo.org=) because =OrgTask.file= goes through
=fs::canonicalize=. After a snapshot was loaded and live reindex ran,
=by_file= ended up with two entries for the same file -- =all_tasks=
walks =by_file= and emits each task once per key, so every task
surfaced twice. Symptom on =visa-nonsoe=: =/api/tasks= returned 460,
=task_count()= was 395.

Fix: =Index::rebuild_file= and friends now canonicalise the input path
before using it as a key, so save/load/rebuild all agree. Regression
test added (=rebuild_via_symlink_does_not_double_count=).

*** Org-spec emphasis markup in titles and bodies

Both =NotesRenderer= implementations now follow Emacs's
=org-emphasis-regexp-components= (boundary chars match the spec; body
forbids its own marker so =*foo*bar*= stays unrendered). Adds
strikethrough (=+text+=) to both clients and underline (=_text_=) to
the web client. Mac task rows now route =task.title= through the same
=renderInline= the body uses, so emphasis lands in headings too.

*** Bridge protocol -- read_exact cancellation fix

The original =BridgeClient::session= used =tokio::select!= over
=read_exact= and =outgoing.recv()=, but =read_exact= is *not*
cancel-safe: when an outgoing write fired between the bytes of an
incoming header, the partial read state was discarded and subsequent
reads desynced. Symptom on =visa-nonsoe=: =read.config= would block
forever despite the bridge actually replying. Fix: separate reader and
writer tasks, neither cancels the other's I/O.

** 0.3.1

Polish + edit-mode rework on top of v0.3.0.

*** macOS task inspector — explicit edit mode

The card no longer drops you into title/notes editing on tap. Edit mode is
opt-in via the new =Edit= action in the row or card context menu. Reading
and editing are now separate states.

- Title is read-only by default; entering edit mode shows the =TextField= and reveals a footer pair of =Discard= (revert original notes + title) and =Save= (=Enter= via =.defaultAction=)
- =originalNotes= / =originalTitle= snapshots taken at load so =Discard= reliably reverts even after multiple in-card edits
- Removed the implicit "tap notes to edit" gesture (=NotesRenderedView= no longer steals taps)
- Removed the redundant =×= close button in the top-right; clicking the title, pressing =Esc=, or clicking outside still dismisses
- Card has its own context menu mirroring the row actions (state / priority / schedule / deadline / clock / refile)

*** Outline breadcrumb in the inspector

A new monospaced =file ▸ Heading ▸ Heading= line under the title row shows where the heading lives. Backed by a new =GET /api/outline?file=&pos== that returns =org-get-outline-path= over the heading's ancestor chain.

- =eav-get-outline-path= elisp helper visits the heading inside =org-with-wide-buffer= and abbreviates the file path (=~/= replaces =$HOME=)
- =APIClient.fetchOutlinePath= + =OutlinePathResponse=; the inspector loads it on appear and on task switch, race-guarded against task changes mid-flight

*** macOS task row — clickable state and priority

The state pill and priority badge are now buttons that open menus directly on the row, no inspector required:

- =stateMenu= reads =keywords= passed in from the parent view and renders =Active= / =Done= sections plus a =Clear= action; arbitrary user TODO sequences flow through unchanged
- =priorityMenu= offers A/B/C/D and =None=
- =keywords: TodoKeywords?= threaded through =MacTaskRow= from =MacTodayView=, =MacUpcomingView=, =MacAllTasksView=, =MacEisenhowerView=, =GroupSection=
- Right-click on a row gains an =Edit= item that calls a new =editInspector= action — opens the card in edit mode in one click

*** macOS Upcoming view — proper day head

Day groups got the same treatment as Today's =dayHead=:

- =TODAY= / =TOMORROW= / =IN N DAYS= / =NEXT WEEK= / =IN N WEEKS= / =IN A MONTH= / =IN N MONTHS= overline, color-graded by proximity
- 22pt bold full date label (=EEEE, MMM d=) with a =N tasks · M events= subtitle

*** Hierarchically-weighted checklist progress

=ChecklistProgress.compute= now builds a tree from indent levels and computes done/ongoing as recursive averages instead of flat counts.

- Avoids double-counting parent items that org auto-aggregates to =[-]=: for the canonical =- [X] A / - [ ] B / - [-] C { - [X] / - [-] }= example the result is =done = 1/2=, =ongoing = 1/6= instead of the old flat =2/5= / =2/5=
- Return type changed from =(done: Int, ongoing: Int, total: Int)= to a =ChecklistProgress= struct with =Double= fractions in =[0, 1]= plus =itemCount= for "% complete" tooltips
- Call sites in =MacTaskRow= and =TaskExpandedCard= use the fractions directly

*** macOS clock dock — robust priority lookup

=priorityFor= now matches a session by id *or* file+title:

- The session id is a =file::pos= snapshot taken at clock-in; as CLOCK lines accumulate in the LOGBOOK drawer the heading's =pos= shifts and the id goes stale. Matching by =(file, title)= as a fallback survives that drift
- Generic =find<T: TaskDisplayable>= helper folds the three identical lookups (=allTasks=, =today=, =upcoming=) into one
- Cosmetic: =⏱= → =⏰= so the dock icon reads at the system size

*** Sidebar dismisses inspector

Clicks anywhere in the =NavigationSplitView= sidebar (rows or empty space) now clear =taskSelection.taskId= via =simultaneousGesture(TapGesture)=. Plain =onTapGesture= wasn't reaching the =NSTableView=-backed sidebar items on macOS.

Replaces the row when selected — clean Things-3-style card, no surrounding border, soft 14pt shadow:

- Dark compact popover replaces the heavy =.graphical= =DatePicker=

*** Row highlight + checklist progress settings

Three new settings (Settings → Row Highlight, Settings → Checklist Progress):

- *Highlight by* — =Off=, =Priority=, or =TODO State= — colors come from =settings.resolvedPriorityColor/resolvedTodoStateColor= so user-customized hexes flow through automatically
- *Highlight style* — =Edge bar= (3pt left strip) or =Background tint= (subtle row fill)

*** Notes parsing + clock data

- =NotesParser= already skipped =:LOGBOOK:= drawers; the inspector hides them too
- Fixed bug in =eav-add-clock-entry=: when a =:LOGBOOK:= drawer existed, the new CLOCK line was being inserted *after* =:END:= instead of inside the drawer (cause: =org-end-of-meta-data= jumps past the drawer; the old code did =forward-line 1= from there). New behavior: jump to the =:LOGBOOK:= header, =forward-line 1=, insert.
- =eav-clock-in= / =eav-clock-out= bind =org-clock-into-drawer "LOGBOOK"= so their CLOCK lines always land inside the drawer regardless of the user's personal config
- New =eav-tidy-clocks= elisp + =POST /api/clock/tidy= server route + =store.tidyClocks= → folds any pre-existing loose =CLOCK:= lines under a heading into a =:LOGBOOK:= drawer
- =eav--collect-loose-clock-lines= helper sweeps loose lines while preserving entry-end positioning via a marker
- Inspector shows an orange =N loose CLOCK entries outside :LOGBOOK:= banner with a =Tidy= button when legacy loose entries are detected

*** =org-todo-list=-correct task filtering

- =eav-extract-all-tasks= was passing =nil= as the match argument to =org-map-entries=, walking every heading. Fixed: only headings with =(org-get-todo-state)= are included.
- =eav-extract-active-tasks= had a similar bug — and was using a hardcoded =("DONE" "KILL" "[X]")= list. Now uses a new =eav--done-keywords= helper that parses =org-todo-keywords= (everything after the =|= per sequence, stripping =(c)= shortcut hints), making it correct for arbitrary user TODO sequences.
- =eav-extract-tasks-for-file= gets the same TODO-state guard.

*** Live progress + observation

- =TasksStore.prefetchNotes= now always re-fetches on row appearance (only the in-flight de-dup set short-circuits) so the row's progress reflects live state even when notes are edited out-of-band in Emacs
- =TasksStore.notesCacheRevision= bumps on every cache mutation; =RowActionFactory.progress(for:)= reads it explicitly so the @Observable framework reliably tracks dictionary mutations through the helper-closure boundary

*** Click-outside / dismiss UX

- Escape key, the × in the inspector top-right, and clicks anywhere outside the card all dismiss the inspector
- Dismiss handler moved from the =ScrollView=' s =.background= (which doesn't extend into scrollable content) to the =LazyVStack=' s background with =minHeight: 600=, so empty space inside the scroll content also dismisses
- Wired in Today / Upcoming / All Tasks / Eisenhower

*** Calendar sidebar regression fix

- Switching to the Calendar view dropped the sidebar's icons + Filters section + footer because the =RootView=' s =HSplitView { mainPane | MacScheduleTray }= nested badly inside the outer =NavigationSplitView=. Moved the schedule-tray split inside =MacCalendarView= as an =HStack { calendarPane | MacScheduleTray }= with a fixed 340pt tray. The detail area is now a single column for every view.
